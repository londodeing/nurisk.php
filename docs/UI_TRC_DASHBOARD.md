# UI Design: TRC Dashboard (Mobile Optimized)

**Target Pengguna:** Tim Reaksi Cepat di lapangan (Smartphone/Tablet).
**Tujuan:** Mencegah TRC tersesat dan mempercepat laju survei (*Assessment*).

## Komponen Layar (Single Page)

```text
[ Navbar: TRC Status: ACTIVE ] [ GPS: OK ]

+-----------------------------------------------------------+
| TUGAS ASESMEN SAAT INI (ACTIVE ASSIGNMENT)                |
| Banjir Bandang - Jembatan Runtuh Desa Cikarang            |
| Jarak: 2 KM | Prioritas: MERAH                            |
| [ TOMBOL: MULAI PENUGASAN ]                               |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| NEARBY INCIDENTS (Belum Tersentuh)                        |
| 1. Longsor Bukit Hijau (1.5 KM)                           |
| 2. Pipa Pecah Jalan Sudirman (3 KM)                       |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| QUICK REPORT (2 Klik)                                     |
| [ AMBIL FOTO ] -> [ PILIH KONDISI: JALAN PUTUS ] -> SEND  |
+-----------------------------------------------------------+
```

## Human Factors
- **Lokasi Pertama:** Peta dan *GPS Status* adalah dewa. TRC bergerak berdasarkan proximitas (jarak), bukan instruksi tertulis yang panjang.
- **Tombol Super Besar:** Tombol *Quick Report* dan Kamera harus seukuran jempol dewasa agar mudah ditekan di atas motor / lumpur.
